This is the marketing/education site for pyportfolios.com. It is a Next.js static-export project: next build emits a fully static site to site/out/, which is then served by any static web server (we use nginx).

Read this first. The active branch is Bhavya — that is where all current work lives. The default branch (main) is intentionally behind. Always check out Bhavya before building.

01Get the code (right branch)

git clone <repo-url> pyportfolios
cd pyportfolios
git checkout Bhavya
git pull origin Bhavya

02Install dependencies

Requires Node.js 18.18+ (Node 20 LTS recommended). From the site/ folder:

cd site
npm ci          # clean, lockfile-exact install (recommended)
# or: npm install

03Build the static site

npm run build   # generates site/out/  (the full themed site + assets)

The complete deployable site now lives in site/out/.

04Run it

GoalCommand / action
Local devnpm run dev → http://localhost:3000 (hot reload)
ProductionPoint your web server's web root at site/out/

For nginx, set the server block's root to the absolute path of …/site/out, serve index.html, then nginx -s reload (or start nginx). Our local reference config is in deploy/.

05What is & isn't in the repo

ItemStatus
Source, configs, fonts (Switzer), images, logosCommitted ✓
package-lock.jsonCommitted ✓ (reproducible installs)
node_modules/Ignored — regenerate with npm ci
site/out/ and .next/Ignored — regenerate with npm run build
"Theme / images missing" after a pull? Two usual causes: (1) you're on main instead of Bhavya, or (2) you haven't run npm ci && npm run build, so site/out/ doesn't exist yet. Do both and it will render fully.
